SOLID PRINCIPLES APPLICATION
================================================================================

Single Responsibility Principle:
  ‚úÖ DataService - Only data operations
  ‚úÖ AIService - Only AI features
  ‚úÖ Parsers - One format each
  ‚úÖ Each module has one clear purpose

Open/Closed Principle:
  ‚úÖ New data sources = new parser (no changes to existing code)
  ‚úÖ Extensible service layer

Liskov Substitution Principle:
  ‚úÖ Database session dependency injection
  ‚úÖ Parser interface compatibility

Interface Segregation Principle:
  ‚úÖ Focused service interfaces
  ‚úÖ Separate schemas for different operations

Dependency Inversion Principle:
  ‚úÖ Services depend on Session abstraction
  ‚úÖ Config management decoupled

DATA INTEGRATION
================================================================================

QuickBooks Data (data_set_1.json):
  - Format: Hierarchical P&L report
  - Coverage: Jan 2020 - Aug 2025 (68 months)
  - Structure: Nested rows with groups
  - Parser: Extracts monthly columns, navigates hierarchy

Rootfi Data (data_set_2.json):
  - Format: Array of monthly records
  - Coverage: Aug 2022 - Jul 2025 (35 periods)
  - Structure: Flat with nested line_items
  - Parser: Recursive line item aggregation

Unified Storage:
  - Single FinancialPeriod model
  - Source tracking for data lineage
  - JSON fields for flexible breakdowns
  - 103 total periods in database
